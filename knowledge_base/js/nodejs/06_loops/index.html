Циклы - for, while, do...while

Мы рассмотрели с вами возможность выполнять разные вещи по условиям. Но что, если мы хотим повторить указанный скрипт определенное количество раз, или еще интереснее, повторять его до наступления какого-то условия.

Тут то нам на помощь и приходят циклы.

Его величество For

Цикл for есть почти во всех языках программирования, и в очень многих из них он выглядит примерно одинаково

for (let i = 0; i < 10; i++) {
  console.log('i', i);
}

Как можно видеть, конструкция for состоит из трех частей:

for (начальное состояние ; проверка ; шаг ) {
  тело цикла
}

начальное состояние - этот кусок выполняется один раз перед началом работы цикла. В нашем случае переменная-счетчик i устанавливается в значение 0

проверка - проверка выполняется перед каждым выполнением цикла, пока она истинна, цикл продолжает повторяться. У нас пока i меньше 10

шаг - это выражение выполняется после каждого прогона цикла. То есть после каждого выполнения тела цикла у нас i увеличивается на 1 (i++ это просто краткий вариант записи i = i + 1)

Итого получаем, что запись вида for (let i = 0; i < 10; i++) {...} обеспечивает выполнение цикла 10 раз

Также легко немного модифицируя числа добиваться разного поведения

// в обратном порядке
for (let i = 10; i > 0 ; i--) {
  ...
}
// с шагом 2
for (let i = 0; i < 10 ; i = i + 2) {
  ...
}

Причем любую из этих частей можно оставить пустой. Но чаще всего цикл for используется именно так. Но возможны и варианты, например:

// без тела
for (let answer = ''; answer != 'Y'; answer = readlineSync.question('Принимаете условия лицензионного соглашения? ')) {
}
// без шага
for (let answer = ''; answer != 'Y'; ) {
  answer = readlineSync.question('Принимаете условия лицензионного соглашения? ');
}
// без начального состояния (используя уже объявленную ранее переменную)
let mult = readlineSync.question('Введите первый множитель ');
for (; mult < 100; ) {
  const n = readlineSync.question('Введите еще один множитель ');
  mult *= n; // эквивалент mult = mult * n;
}
// вечный цикл
for (; true ; ) {
  ...
}

Но такие варианты не всегда удобочитаемы, поэтому чаще всего for используется именно в виде for (let i = 0; i < 10; i++) {...}

// Существуют еще две формы цикла for - for..in и for..of, но их мы обсудим позднее, когда перейдем к изучению массивов и объектов

While

Довольно часто нам нужно просто выполнять цикл до наступления какого-то условия, и в этом случае конструкция с for будет немного избыточной и громоздкой.

Для таких случаев был придуман цикл while

let n = 1;
while (n % 2 != 0) { // n % 2 - это остаток от деления на 2, то есть он будет равен 0 для любого четного числа и самого нуля.
  n = readlineSync.question('Введите четное число ');
}

Он будет выполняться до тех пор, пока условие в скобках истинно, причем эта проверка проходит перед выполнением тела цикла. То есть если выражение изначально ложно, то тело не будет выполнено совсем.

// Обратите внимание, в таком случае можно легко упустить изменение проверяемого значения, и превратить цикл в вечный.

// простейший вечный цикл
while (true) {
  ...
}

Техника безопасности: если вы случайно запустили вечный цикл, то остановить выполнение программы можно нажав клавиши Ctrl+C

do...while

Если в цикле while первоначальное значение ложно, то цикл не выполнится ни разу. Обратите внимание, как в случае с четными числами чуть выше мы устанавливали первоначальное значение n так, чтобы цикл не вылетел сразу.

Упростить себе этот момент можно используя цикл do...while.

let n;
do {
  n = readlineSync.question('Введите четное число ');
} while (n % 2 != 0);

do...while выглядит немного непривычно и используется не очень часто, но выручает, когда тело цикла должно обязательно выполниться хотя бы раз.

Задачи

  • Используя цикл for выведите в консоль все числа от 0 до 20

  • Усовершенствуйте предыдущий скрипт, чтобы он выводил только четные числа

  • Усовершенствуйте предыдущий скрипт, чтобы он спрашивал у пользователя число, а потом выводил все числа до 100 делящиеся на это число.

  • Создайте переменную с каким то произвольным значением пароля и при запуске вашего приложения не пускайте пользователя дальше, пока он не введет требуемый пароль.
