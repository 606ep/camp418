Переменные в JavaScript

Как и во многих других языках программирования, переменные в JS это "ячейки", в которых мы можем хранить информацию.

Для создания переменной используется ключевое слово let

let a; // Создаем переменную 'a'
let b = 1; // Создаем переменную 'b' со значением 1
a = 2;  // Присваиваем 'a' новое значение
b = 15; // Присваиваем 'b' новое значение

Что можно делать с переменными

Переменным можно присваивать не только конкретные значения, но и выражения, например арифметические (по правде говоря везде где используется значение, можно использовать вместо него выражение).

let a = 18 + 22;
let b = a * 44;
let c = 'Hello';
console.log(a); // выводим значение перменной `а` в консоль

Правила именования

В js есть несколько правил по именованию переменных. Название переменной должно:

  • содержать только буквы (кстати любого алфавита), цифры или символы $ и _
  • не начинаться с цифры
  • не быть зарезервированным словом JS
// валидные (допустимые) названия
let a = 1;
let A = 2;
let letItBeAVeryLongName = 'ok';
let another_very_long_name_of_variable = 'allowed';
let b812 = 999;
// невалидные (недопустимые) названия
let 8b = 1;   // начинается с цифры
let let = 12; // Зарезервированное слово

Обрати внимание, что в JS учитывается регистр в названиях. То есть someVARIABLE и someVariable - будут двумя совершенно разными переменными.

Также не обязательно, но крайне желательно начинать названия перменных с маленькой буквы. Названия начинающиеся с заглавной буквы чаще принято давать классам.

Переменные и константы

Помимо переменных мы можем создать константу - такую ячейку, значение в которой не будет изменяться никогда после инициализации. Делается это при помощи ключевого слова const

let a = 1;
const b = a + 1;
console.log(a); // выведет 1
console.log(b); // выведет 2
a = 12; // пройдет нормально
b = 33; // выдаст ошибку

Подходы к наименованию переменных

Важно: всегда называйте переменные понятными именами. Однобуквенные названия кажутся удобными поначалу, но нанесут непоправимый урон вашей психике, при попытке через пару дней (или часов) понять, а что же делает ваш код.

Довольно часто хорошее название переменной (из которого понятно, что в ней хранится) должно состоять из двух и более слов. Существует несколько подходов к написанию таких имен:

  • сamelCase - каждое следующее слово начинается с заглавной буквы. countOfRedDots averageSpeed и т.д.

  • snake_case - слова в названии разделены символом подчеркивания. initial_word base_height_value

  • kebab-case - слова в названии разделены дефисом. speed-limit word-count

Какой из варинатов выбрать - непринципиально. Главное придерживаться единого стиля в рамках проекта. Но чаще всего в JS используют camelCase.
